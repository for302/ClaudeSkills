## 1. 연구 목적 및 배경 (Purpose & Background)

### 핵심 문제 (Core Problem)
HPT(Human Performance Technology) 분야가 조직 효율성 향상에 대한 주류의 관심을 받기 시작하면서, 이러한 개념과 이론을 실무에 어떻게 적용할 것인가에 대한 질문이 제기되었다. HPT 실무자들은 다양한 모델들 중에서 어떤 것을 선택하고 적용해야 할지에 대한 명확한 가이드가 필요했다.

### 기존 연구의 한계 (Limitations of Existing Research)
- 초기 HPT 실무자들이 선형적인 교수설계 모델을 성과 기술 프로세스에 적용하려 했으나, 이는 복잡한 비즈니스 프로세스의 환경이나 상호관계를 정확히 설명하지 못함
- HPT 분야의 선구자들이 각자의 관점과 배경에 따라 독자적인 모델을 개발하여 모델의 다양성과 복잡성이 증가
- 모델들의 체계적인 분류와 비교가 부족하여 실무자들이 적절한 모델을 선택하기 어려움
- 훈련 격차(training gap)와 성과 격차(performance gap)의 구분이 불명확

### 본 논문의 해결 방향 (Proposed Solution Direction)
본 논문은 HPT 분야의 주요 모델들을 확인하고, 각 모델의 개념적 기초, 개발 과정, 수용 이유를 체계적으로 검토한다. 모델들을 **진단 모델(Diagnostic Models)**, **프로세스 모델(Process Models)**, **통합 모델(Holistic Models)**의 세 가지 범주로 분류하여, 실무자들이 상황에 맞는 모델을 선택할 수 있도록 돕는다.

## 4. 연구 도구 및 시스템 (System or Model)

### HPT 모델 분류 체계 (Classification Framework)

본 연구에서 제시한 HPT 모델 분류 체계는 다음과 같다:

#### 1. **진단 모델 (Diagnostic Models)**
- **목적**: 성과 문제가 **어디에** 있는지 파악
- **특징**: 문제의 위치 식별에 초점
- **대표 모델**:
  - **Deterline's Performer-Centered Model**: 개인 성과자에게 영향을 미치는 다양한 요인 분석
  - **Wile's Synthesized HPT Model**: 외부(환경/자원)와 내부(기술/지식) 도메인으로 구분
  - **Tosti & Jackson's Organizational Scan Model**: 조직, 사람, 업무의 세 가지 수준에서 성과 영향 분석
  - **Langdon's Language of Work Model**: Input → Process → Output → Consequences의 흐름으로 성과 분석

#### 2. **프로세스 모델 (Process Models)**
- **목적**: HPT 솔루션을 **어떻게** 적용할지 제시
- **특징**:
  - 대부분 선형적(sequential)
  - 단계별/그룹화된 활동 포함
  - 성과 격차(gap) 분석 중심
  - 개입(intervention) 지향적
  - 피드백 메커니즘 포함
- **대표 모델**:
  - **ISPI Model**: 성과 분석 → 원인 분석 → 개입 선택 및 설계 → 실행 및 변화 관리 → 평가
  - **Human Performance Model**: 전략 및 이슈 → 격차 분석 → 개입 영역 → 개입 → 측정
  - **Harless's ADDIE Model**: Analysis → Design → Development & Testing → Implementation & Evaluation

#### 3. **통합 모델 (Holistic Models)**
- **목적**: 비선형적이고 통합적인 접근
- **특징**: 중첩된 도메인, 생태계적 관점
- **대표 모델**:
  - **Advancia's Holistic Model**: People, Processes, Organization의 세 개 원이 겹쳐진 구조
  - **Stock's 3-Dimensional HPT Model**: Rational, Emotional, Executive/Managerial/Performer 차원의 3차원 모델 (감성 지능 포함)

### 주요 선구자 모델

#### **Thomas Gilbert's Behavior Engineering Model**
- 성과 향상은 환경 장벽을 식별하고 제거하는 것에서 시작
- "Father of Performance Technology"

#### **Joe Harless's Front-End Analysis Model**
- 문제의 원인 파악이 해결책을 주도해야 함
- 초기 Performance Technology Process I → 후기 ADDIE 모델로 발전

#### **Robert Mager's Models**
- **Instructional Objectives Model**: Performance, Conditions, Criterion 세 요소
- **Performance Analysis Flow Chart**: 성과 문제 식별 및 해결 단계 제시

#### **Geary Rummler's Nine Performance Variables**
- 조직을 생태계로 간주 (모든 요소가 상호 연결)
- 세 가지 수준: Organizational level, Process level, Job/Performer level
- 9개 성과 변수: Goals, Design, Management 범주로 구분

## 5. 주요 결과 (Findings)

### RQ1 결과: HPT 분야의 주요 모델
**Finding**: HPT 분야에는 다양한 모델이 존재하며, 선구자들(Gilbert, Harless, Mager, Rummler)의 기여를 기반으로 발전해왔다.

- **Evidence**: 논문에서 15개 이상의 HPT 모델을 확인하고 분석함
- **Significance**: 단일 모델이 아닌 다양한 접근 방식이 필요함을 시사

### RQ2 결과: 모델 분류 체계
**Finding**: HPT 모델은 세 가지 주요 범주로 분류 가능: Diagnostic, Process, Holistic

- **Evidence**:
  - **Diagnostic models**: 문제 위치 파악 (예: Wile's model, Langdon's model)
  - **Process models**: 솔루션 적용 방법 (예: ISPI model, ADDIE)
  - **Holistic models**: 통합적 접근 (예: Stock's 3D model)
- **Significance**: 실무자가 상황에 맞는 모델 선택 가능

### RQ3 결과: 프로세스 모델의 공통 특성
**Finding**: 프로세스 모델은 5가지 공통 특성을 가짐

- **Evidence**:
  1. Linear/Sequential 구조
  2. Phased or grouped activities
  3. Gap analysis 중심
  4. Intervention-oriented
  5. Feedback mechanism 포함
- **Significance**: 체계적이고 측정 가능한 성과 개선 프로세스 제공

### RQ4 결과: HPT 성공의 핵심 요소
**Finding**: Front-end analysis, Measurement, Experience가 HPT 성공의 3대 핵심 요소

- **Evidence**:
  - **Front-end analysis** (Harless): 원인 파악이 솔루션을 주도
  - **Measurement** (Mager): 명확하고 측정 가능한 성과 목표 필요
  - **Experience**: 다양한 전문성을 가진 팀 필요
- **Significance**: 실무 적용 시 이 세 요소를 반드시 고려해야 함

## 6. 논의 및 시사점 (Discussion & Implications)

### 이론적 기여 (Theoretical Contributions)
1. **HPT 모델의 체계적 분류 프레임워크 제공**
   - Diagnostic, Process, Holistic 삼분법은 모델의 목적과 특성을 명확히 구분
   - 후속 연구자들이 새로운 모델을 개발하거나 평가할 때 참조할 수 있는 기준 제시

2. **HPT 분야의 역사적 발전 과정 정리**
   - 교수설계(ID)에서 HPT로의 패러다임 전환 과정 추적
   - 훈련 중심 → 성과 중심으로의 사고 전환 강조

3. **개인 성과와 조직 성과의 연결**
   - Rummler의 기여를 통해 개인과 조직 수준의 성과가 별개가 아니라 상호 연결됨을 강조

### 실무적 시사점 (Practical Implications)

**For HRD Professionals**:
- 성과 문제가 항상 훈련(training)으로 해결되는 것은 아님을 인식
- Front-end analysis를 통해 진짜 문제의 원인 파악 필요
- 상황에 맞는 모델 선택: 문제 진단이 필요하면 Diagnostic, 프로세스 가이드가 필요하면 Process 모델 활용

**For Performance Consultants**:
- 클라이언트와의 파트너십 중요 (Harless의 강조점)
- 측정 가능한 성과 목표 설정 (Mager의 기여)
- 다학제적 팀 구성 필요 (한 사람이 모든 분석을 수행하기 어려움)

**For Organizational Leaders**:
- HPT는 훈련 이상의 포괄적 접근
- 환경적 요인(도구, 프로세스, 보상 등)이 개인 성과에 미치는 영향 고려
- 조직 수준, 프로세스 수준, 개인 수준의 성과를 동시에 고려해야 함

### 분야별 적용 (Field-Specific Applications)

**HRD (Human Resource Development)**:
- 직원 성과 문제 진단 시 Wile's Synthesized Model 활용 가능
- 교육 프로그램 개발 시 ADDIE 프로세스 모델 적용
- ROI 측정을 위한 명확한 성과 기준 설정 (Mager의 접근)

**조직 개발 (Organizational Development)**:
- Rummler의 Nine Performance Variables로 조직 전체 진단
- Tosti & Jackson의 Organizational Scan으로 조직-프로세스-개인 수준 통합 분석

**교육 설계 (Instructional Design)**:
- ADDIE 모델을 기반으로 하되, 비훈련 개입(non-training interventions)도 고려
- Langdon's Language of Work로 학습자의 업무 흐름 파악

## 7. 디자인 한계 (Design Limitations)

### 연구 설계 한계 (Research Design Limitations)
- **문헌 연구의 한계**: 실증적 데이터나 사례 연구가 부족하여 각 모델의 실제 효과성을 비교하지 못함
- **2014년 기준**: 최신 모델이나 디지털 전환 시대의 HPT 접근은 포함되지 않음

### 데이터 한계 (Data Limitations)
- **모델 효과성 비교 부재**: 어떤 모델이 어떤 상황에서 더 효과적인지에 대한 실증 데이터 없음
- **산업별/조직별 차이 미고려**: 특정 산업이나 조직 규모에 따른 모델 적합성에 대한 구체적 분석 부족

### 일반화 가능성 (Generalizability)
- 주로 북미 지역의 HPT 실무와 문헌에 기반
- 문화적 맥락이나 국가별 차이 고려 부족
- 비영리 조직, 정부 기관 등 다양한 조직 유형에 대한 적용 가능성 명시 부족

### 실무 적용 한계 (Practical Application Limitations)
- 모델 선택을 위한 구체적인 의사결정 도구나 체크리스트 미제공
- 하이브리드 모델(여러 모델 결합) 사용에 대한 가이드 부족
- 실무자의 역량 수준에 따른 모델 선택 가이드 부재

## 9. 연구의의 (Significance)

### 학문적 기여 (Academic Contribution)
본 연구는 HPT 분야의 주요 모델들을 체계적으로 정리하고 분류함으로써 분야의 이론적 기초를 공고히 했다. 특히 Diagnostic, Process, Holistic의 삼분법 분류 체계는 후속 연구자들이 새로운 모델을 개발하거나 기존 모델을 평가할 때 유용한 준거 틀을 제공한다. 또한 Gilbert, Harless, Mager, Rummler 등 선구자들의 기여를 역사적으로 추적함으로써 HPT 분야의 지식 계보를 명확히 했다.

### 실무적 의의 (Practical Significance)
HRD 실무자들이 조직의 성과 문제에 직면했을 때, 이 연구가 제시한 분류 체계와 모델 개요를 통해 상황에 적합한 모델을 선택할 수 있다. 특히 "훈련으로 모든 문제를 해결할 수 없다"는 인식을 확산시키고, 환경적 요인, 동기, 프로세스 등 다양한 성과 영향 요인을 고려하도록 유도한다. 또한 Front-end analysis, Measurement, Experience라는 세 가지 성공 요소를 제시하여 실무 적용 시 핵심을 놓치지 않도록 돕는다.

### HRD 현장 적용 (Application to HRD Practice)
- **Performance Consulting 강화**: 단순 교육 제공자에서 성과 컨설턴트로의 역할 전환 촉진
- **데이터 기반 의사결정**: 측정 가능한 성과 목표 설정의 중요성 강조
- **체계적 접근**: Ad-hoc 해결책이 아닌 체계적인 성과 분석 및 개입 설계
- **클라이언트 파트너십**: 조직 리더와의 긴밀한 협력을 통한 지속 가능한 성과 개선
